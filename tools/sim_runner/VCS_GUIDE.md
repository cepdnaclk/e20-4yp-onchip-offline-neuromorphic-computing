# VCS & PrimePower Guide — University Server
## Neuromorphic Accelerator: MNIST Inference on Synopsys Tools

---

## Overview

Once you have verified the design locally with Icarus Verilog, you can run
the same testbench on the university server using:
- **VCS** (Synopsys VCS) — faster, industry-grade Verilog simulation
- **PrimePower** — post-simulation power analysis

---

## Part 1: Running with VCS

### 1.1 Transfer Files to Server

Copy these files to your home directory on the server:

```bash
# From your local machine (replace user@server with your credentials)
scp -r inference_accelarator/ user@server:~/fyp/
scp -r shared_memory/         user@server:~/fyp/
scp -r tools/                 user@server:~/fyp/

# Transfer the generated memory files
scp inference_accelarator/neuron_accelerator/data_mem.mem   user@server:~/fyp/inference_accelarator/neuron_accelerator/
scp inference_accelarator/neuron_accelerator/spike_mem.mem  user@server:~/fyp/inference_accelarator/neuron_accelerator/
scp inference_accelarator/neuron_accelerator/test_labels.txt user@server:~/fyp/inference_accelarator/neuron_accelerator/
```

### 1.2 Setup VCS Environment on Server

```bash
# On the server — load VCS (exact module name may differ, ask your admin)
module load synopsys/vcs/latest     # or: module load vcs/U-2023.03
module load python/3.9

# Verify
vcs -version
```

### 1.3 Compile with VCS

```bash
cd ~/fyp/inference_accelarator/neuron_accelerator

# Standard compile (functional simulation)
vcs -full64 -sverilog -debug_access+all +v2k \
    neuron_accelerator_tb.v \
    -o simv \
    2>&1 | tee compile.log

# Check for errors
grep -i "error" compile.log | head -20
```

### 1.4 Run Simulation

```bash
# 100 samples, 25 timesteps (recommended first run)
./simv \
    +time_step_window=25 \
    +input_neurons=784 \
    +nn_layers=2 \
    +input_count=100 \
    2>&1 | tee sim_100_t25.log

# 960 samples (full test set subset), 25 timesteps
./simv \
    +time_step_window=25 \
    +input_neurons=784 \
    +nn_layers=2 \
    +input_count=960 \
    2>&1 | tee sim_960_t25.log
```

### 1.5 Decode and Get Accuracy

```bash
# After simulation completes
python3 ~/fyp/tools/decoder/decode_output.py \
    --sim_output output.txt \
    --labels     test_labels.txt \
    --samples    100 \
    --save_csv   results_vcs_t25.csv
```

## Part 2: Power Analysis with PrimePower

PrimePower requires:
1. A **gate-level netlist** (from synthesis with Design Compiler)
2. A **switching activity file** (SAIF) generated by VCS simulation
3. A **timing library** (.db file) for your target technology

### 2.1 Generate SAIF from VCS Simulation

Recompile with SAIF dumping enabled:

```bash
vcs -full64 -sverilog -debug_access+all +v2k \
    +define+SAIF_DUMP \
    neuron_accelerator_tb.v \
    -o simv_saif

# Run simulation with SAIF power annotation
./simv_saif \
    +time_step_window=25 \
    +input_neurons=784 \
    +nn_layers=2 \
    +input_count=100 \
    +fsdb+all=on \
    -power \
    2>&1 | tee sim_saif.log

# This generates: verilog.saif (switching activity)
```

Add this to your testbench for SAIF capture (if not already present):

```verilog
// In neuron_accelerator_tb.v initial block:
initial begin
    $set_gate_level_monitoring("off");
    $set_toggle_region(uut);  // uut = your DUT instance name
    $toggle_start();
    // ... rest of simulation ...
    $toggle_stop();
    $toggle_report("activity.saif", 1.0, "uut");
end
```

### 2.2 Design Compiler Synthesis (Optional — for gate-level netlist)

If you want a technology-mapped netlist for realistic power numbers:

```tcl
# dc_shell synthesis script (synthesis.tcl)
set_app_var search_path ". /path/to/tech/library"
set_app_var target_library "your_tech.db"
set_app_var link_library "* your_tech.db"

read_verilog neuron_accelerator.v
current_design neuron_accelerator
link

# Set clock
create_clock -period 10 [get_ports clk]   ;# 100 MHz

# Compile
compile_ultra
write -format verilog -output neuron_accelerator_netlist.v
write_sdf neuron_accelerator.sdf
write_sdc neuron_accelerator.sdc
```

### 2.3 Run PrimePower

```tcl
# primepower.tcl
set_app_var search_path ". /path/to/tech/library"
set_app_var target_library "your_tech.db"
set_app_var link_library "* your_tech.db"

# Read gate-level netlist
read_verilog neuron_accelerator_netlist.v
current_design neuron_accelerator
link

# Apply constraints
read_sdc neuron_accelerator.sdc

# Read switching activity (SAIF)
read_saif -input activity.saif -instance uut

# Compute power
update_power
report_power > power_report.txt
report_power -hierarchy >> power_report.txt

echo "Power analysis complete. See power_report.txt"
```

Run it:

```bash
module load synopsys/primepower/latest
pt_shell -f primepower.tcl 2>&1 | tee primepower.log
```

## Part 3: Results to Include in Presentation / Paper

After running the above, you will have:

### Accuracy Results
- Top-1 accuracy on MNIST test set (hardware inference vs. software)
- Per-class accuracy breakdown (0–9 digits)
- Effect of timesteps T on accuracy (run T=25, 50, 75, 100)

### Power Results (from PrimePower)
- Total dynamic power (mW)
- Static / leakage power
- Per-module power breakdown (spike network, neuron cluster, weight resolver)
- Energy per inference = Power × (simulation_cycles / clock_frequency)
